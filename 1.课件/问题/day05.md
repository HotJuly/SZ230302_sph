# 1.Vue项目中,静态src和动态src的区别

```
静态src
	在书写组件的template模版的时候,输入字符串,写死src标签属性
		没有任何动态数据的参与
		
动态src
	在书写组件的template模版的时候,img标签的src属性,不是写死的字符串
		而是使用data等响应式数据书写的动态属性
		
解析流程区别:
	静态src
		在项目以生产环境打包的过程中,脚手架如果发现某个img标签,使用src引入了某个静态资源
			那么脚手架就会自动将该资源携带上,最终输出到成品项目的images文件夹中
			
	动态src	
		在项目以生产环境打包的过程中,脚手架如果发现某个img标签,使用src配合状态数据书写
			他不会携带上这个资源
			
		原因:因为在打包项目的时候,脚手架看到的就是:src="src",他只知道src是一个动态数据,不知道src的具体值是什么,因为想要知道src的状态值,只有当用户使用该项目,组件挂载之后,执行组件的data函数的时候,才能知道src的值是多少,但是此时项目已经错了打包的时间,打包的时候没有带上这张图片
```
